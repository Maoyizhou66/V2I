# 面向试车场多工况场景的车路协同V2I融合技术：基于卡尔曼滤波与高效注意力机制的联合状态估计
车-路-云 决策级融合
# 前言
自动驾驶技术尽管在近年来取得了长足进步，但其在全局态势感知、远程感知和盲区感知等方面仍存在显著局限，构成当前系统安全性的关键瓶颈。随着路测基础设施的逐步部署与性能提升，融合车载传感器与路端感知单元的多源信息，能够有效弥补单车智能的不足，包括提供超视距感知范围，消除盲区，提供精确的车道级目标与运动态势信息。因此，车路协同V2I正逐渐成为自动驾驶演进的重要方向，而封闭/半封闭试车场、港口、矿区等限定场景成为该技术率先实现工程化落地的主要应用场景。南宁试车场的反季节低温耐久路试中，高速封闭路段的感知距离不足，加上环路本身视野狭窄，两个问题叠加，导致基础碰撞安全存在明显短板。针对上述问题，本文聚焦于比亚迪南宁反季节试验场（亚洲唯一此类专用场地）这一典型限定场景，从相机部署方式与观测视角出发，将纯视觉BEV感知系统划分为三个层级：车端单视角感知、路端固定视角感知，以及全局视角感知。三路感知信号最终在车端进行决策级融合，从而形成车路协同感知下的统一态势认知，为安全策略的制定与执行提供可靠支撑。
# 1.研究现状
车路协同感知的数据粒度和通信带宽之间存在平衡，现有方案分为三个层级。数据级协同，智能网联汽车从其他智能体处接收原始感知数据并执行时空校准，然后进行多源数据的融合。该协同策略能获取高保真信息，从根源上解决远距离感知和盲区遮挡问题，但伴随着海量的数据传输和精确同步的硬性要求，在实际工程化中难度很大。特征级协同，各智能体将原始感知数据经本地编码网络提取为特征向量再进行传输，在智能网联汽车端进行特征融合和推理。该策略显著降低带宽需求，有利于实时性保证，但是特征编码会不可避免引入信息损失和噪声，且各智能体的模型会带来特征空间不匹配的问题，给融合算法设计带来显著挑战。决策级协同，仅共享各智能体独立的感知结果(如目标检测框、分类标签或轨迹)，智能网联汽车端进行结果级融合。该策略通信带宽被降至最低，且各智能体解耦，适合工程落地，但是观测信息的细粒度在逐级压缩中逐渐丢失，且性能高度依赖各智能体的独立能力。综上所述，不同车路协同策略对带宽(传输速率)和检测精度以及数据颗粒度如表1所示。聚焦于试车场封闭式低温耐久环路，数据级协同虽然信息保真度最高，但车端接收点云和图像数据可达几百兆/秒，而试车场5G专网的带宽有限，且车辆较多，难以稳定支撑数据传输。特征级协同，虽然在理论层面兼顾了通信效率和信息颗粒度，但不同智能体所搭载的特征提取模型往往不一致，特征空间对齐也需要额外的标定环节。相比之下，决策级协同是当前工程约束下唯一可行的方向，通信开销最低，路侧感知结果可直接与车端已有检测输出进行融合，无需修改各智能体原有架构，但现有决策级方案在封闭式环路场景中会存在时序信息缺失和感知精度波动的问题，因此需要进行针对性改进。鉴于此，需研发一种面向封闭式、高速、环路场景的决策级融合算法。在保留决策级协同低通信开销优势的前提下，针对时序信息缺失问题，在路端与车端交互的检测结果中附加运动矢量；针对时空不同步问题，采用互相关时延估计进行时间对齐，通过轨迹形状匹配，实现空间对齐；针对感知精度波动问题，构建基于Hausdorff距离的动态轨迹匹配与融合机制，有效应对数据异步和局部轨迹波动较大。进一步地，将反季节场地按空间位置划分为直路、低环、高环三种场景，采用交互式多模型(IMM)滤波器(直路优选匀速模型，弯道优选转弯模型)，根据目标运动状态切换融合策略，最终输出高可靠性、低延迟的融合目标，供规划控制决策使用。
# 2.单智能体感知
本文重点关注车辆-路端基础设施的协作，车辆接收并整合来自路端的信息，以定位和识别周围的物体。
# 2.1 车端单视角感知
车端单视角感知采用BEV方法，基本上都遵循着如下范式：首先将多目环视相机的图像特征从二维图像空间转换至车身坐标系下的三维BEV特征空间，然后采用特定的head结构应用于统一的BEV特征，以便执行定制的3D任务，统一的BEV特征可以高效且灵活的执行单个任务或同时执行多项任务，例如：3D 检测、分割等多种任务。基于Transformer的BevFormer、基于深度转换的BevDepth等对算力要求较高，而Fast-BEV是一种对于车载芯片友好且简单轻便的BEV框架，且能适配黑芝麻平台C1200芯片。
